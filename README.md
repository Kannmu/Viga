# 产品需求文档 (PRD)：Viga( **V**ector **I**ntelligence **G**raphic **A**ssistant) - 开源AI驱动协作设计软件

## 1. 产品概述 (Product Overview)

### 1.1 产品简介

Viga 是一款开源的、AI驱动的人机协作矢量设计软件。它结合了传统专业设计软件（如Figma）的全套手动矢量编辑能力，同时原生内嵌大语言模型（LLM）能力。支持用户通过自然语言从零生成矢量设计稿，并能让AI读取、理解和修改现有设计稿。

### 1.2 目标用户与核心场景

* **科研工作者**：通过自然语言描述实验流程或数据逻辑，快速生成论文级别的标准矢量配图，并能手动微调导出高质量PDF/SVG。
* **网页/前端UI设计师**：通过 Prompt 快速生成网页线框图、UI组件库，并在画布上与AI对话修改布局、配色，随后手动精细化调整。
* **矢量绘图师/插画师**：AI辅助生成基础图形轮廓或复杂图案排版，绘图师利用专业钢笔工具和布尔运算进行艺术化创作。

### 1.3 核心价值主张

* **100% 掌控权**：大模型服务完全自定义（BYOK），不绑定特定厂商。
* **无缝人机协作**：AI不只是生成图片的“黑盒”，而是生成可编辑的“矢量图层”，人类可以随时接管修改。
* **专业级编辑体验**：对标 Figma 的交互和功能标准，降低设计师的学习成本。

## 2. 功能需求详细说明 (Functional Requirements)

### 2.1 AI 模型配置与管理模块 (Model Configuration)

此模块旨在让用户能够自由接入任何支持 OpenAI API 格式的大语言模型（如 GPT-4, Claude-3 甚至本地部署的 Llama-3、Qwen 等）。

* **配置项输入**：
  * **API Key**: 允许输入并安全存储用户的 API Key。
  * **Base URL**: 允许自定义 API 的请求网关地址。
  * **Model Name**: 允许用户手动输入或从接口读取支持的模型名称。
* **多配置管理**：支持保存多套模型配置方案（如一套用于生成UI，一套本地模型用于处理机密科研图纸），允许用户一键切换。
* **连通性测试**：提供“Test Connection”按钮，一键验证配置是否可用。

### 2.2 AI 驱动设计模块 (AI-Driven Design)

* **文本到矢量生成 (Text-to-Vector)**：
  * 用户可以在全局指令框中输入 Prompt，AI 在画布指定区域生成完全可编辑的矢量图层组合（而非位图）。
* **上下文感知与读取 (Context Awareness)**：
  * AI 能够“看懂”当前画布或被选中图层的内容、结构、属性（如位置、颜色、圆角、文本内容）。
* **局部/对话式修改 (Contextual Modification)**：
  * 选中一个或多个设计元素，唤出 AI 助手，通过对话修改该元素。例如：“将这些按钮的主色调改为深色模式”、“把这个流程图整理对齐，并把所有箭头加粗”。
* **历史版本与差异对比 (Diff & Versioning)**：
  * AI 的每次修改都应作为一个独立的动作。提供类似代码合并的“预览/接受/拒绝”机制。如果AI改错了，用户可以一键撤销到上一步手动状态。

### 2.3 核心矢量绘图模块 (Core Vector Editing - 严格对标 Figma)

系统必须提供一套完整、手动、非破坏性的矢量编辑工具链。

* **基础绘图工具**：矩形、圆形、多边形、星形、直线、箭头等。
* **高级矢量工具 (Vector Networks)**：支持专业级钢笔工具（Pen Tool）、贝塞尔曲线调整、多节点连接。
* **属性检查器 (Properties Panel)**：
  * **填充与描边 (Fill & Stroke)**：纯色、线性渐变、径向渐变，描边粗细与对齐方式。
  * **排版 (Typography)**：字体选择、字号、行高、字间距、对齐方式。
  * **效果 (Effects)**：阴影（Drop Shadow）、内阴影、图层模糊、背景模糊。
* **层级与组织 (Layer & Organization)**：
  * 图层树面板（Layer Tree）、编组（Group）、画板/容器（Frame）、蒙版（Mask）、布尔运算（交、并、补、差）。
* **布局与对齐 (Layout)**：
  * 网格/标尺（Rulers & Guides）、自动布局（Auto Layout：对应 Flexbox 逻辑）、基础对齐与分布工具。

### 2.4 文件与协作模块 (File & Collaboration)

* **多格式导入/导出 (Import/Export)**：
  * **导出格式**：SVG（主要矢量资产）、PDF（科研论文刚需）、PNG/JPEG（普通预览）。
  * **导入格式**：支持拖拽导入现有的 SVG 文件，并能自动解析为画布上的可编辑图层。
* **项目与文件管理**：
  * 本地/云端文件保存、文件命名、最近打开的文件列表。
* **画布交互操作**：
  * 无限画布、鼠标滚轮缩放/平移、快捷键（必须高度兼容 Figma 的默认快捷键体系，如 V 选中、P 钢笔、T 文本等）。

## 3. 用户操作流示例 (User Workflows)

**场景 1：科研工作者制作论文流程图**

1. 用户在 Viga 中配置好自己的本地/私有化大模型 API 参数（保障科研数据不外泄）。
2. 在 AI 对话框中输入：“帮我画一个神经网络前向传播的流程图，包含输入层、两个隐藏层和输出层，风格极简，适合黑白打印”。
3. AI 思考后，在画布上生成由几十个矩形、文本和连线构成的“矢量图层组”。
4. 用户发现隐藏层的某个节点颜色太深，直接选中该节点，手动通过右侧“属性面板”将填充色调浅。
5. 调整完毕后，框选整体，导出为高质量 PDF 文件插入到 LaTeX 论文中。

**场景 2：UI前端设计师构建后台界面**

1. 连网状态下调用 GPT-4 模型，输入：“生成一个现代化 SaaS 系统的左侧导航栏和顶部标题栏”。
2. AI 生成后，设计师发现导航栏缺少一个“设置”图标。
3. 设计师框选侧边栏，呼出 AI：“在这个导航栏底部添加一个‘设置’菜单项”。AI 自动识别当前布局，生成对应文本和矢量齿轮图标，并保持原有对齐方式。
4. 设计师手动使用“Auto Layout”功能调整边距，最后导出为 SVG 组件。

## 4. 非功能需求 (Non-Functional Requirements)

### 4.1 安全性与隐私 (Security & Privacy)

* **密钥安全**：用户的 API Key 和 Base URL 绝对不能上传至软件方服务器。在桌面端/浏览器端必须采取本地加密存储（如 LocalStorage 或系统钥匙串），确保用户自带密钥（BYOK）的安全性。
* **数据隔离**：AI 对画布内容的读取必须是“按需”的。只有在用户主动触发 AI 生成或修改时，才将必要的设计结构数据（如选中元素的 JSON 结构）发送给大模型，防止全局设计资产泄露。

### 4.2 性能与扩展性 (Performance & Extensibility)

* **大文件渲染性能**：即便在画布上存在数万个矢量节点，缩放和平移也需要保持在 60fps 的流畅度。
* **插件与扩展机制**：作为开源软件，架构设计需预留插件接口。无论是新增一种第三方模型的适配逻辑，还是增加一种全新的绘图工具，开发者都能通过编写模块化插件来实现。

### 4.3 开源与本地化支持 (Open Source & Localization)

* **多语言支持 (i18n)**：界面及 AI 预设提示词需支持多语言（首发支持中/英文）。
* **跨平台**：由于用户受众包括科研人员（多用 Windows/Linux）和设计师（多用 Mac），软件必须支持跨平台运行（Web端或跨平台桌面端均可）。
